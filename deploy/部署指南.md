## 1. 架构总览

```
Windows PC（本机）
  ├── WeChat PC — 正常登录使用
  │        ↓ HTTPS 流量经过本地代理
  ├── mitmproxy（端口 8080）— 拦截微信流量，提取文章
  │        ↓ POST http://localhost:3000/api/ingest
  ├── Next.js（端口 3000）— 网页前端 + API
  │        ↓ http://localhost:8001/classify
  └── FastAPI（端口 8001）— AI 分类服务
           ↓
       SQLite（prisma/dev.db）— 本地数据库
```

三个服务均在本机运行，互相通过 localhost 通信，不对外暴露任何端口。

## 2. 前置要求

在安装之前，请确保已安装以下软件：

### Node.js 20+

前往 [https://nodejs.org/zh-cn/download](https://nodejs.org/zh-cn/download) 下载 LTS 版本，
按默认选项安装即可。

验证：
```
node -v   # 应显示 v20.x.x 或更高
```

### Python 3.9+

前往 [https://www.python.org/downloads/](https://www.python.org/downloads/) 下载最新版。

**安装时务必勾选 "Add Python to PATH"**（默认不勾选，不勾选会导致后续报错）。

验证：
```
python --version   # 应显示 Python 3.9.x 或更高
```

### 微信 PC 版

使用 Windows 桌面版微信，并登录你的微信账号。需要订阅你希望收录的公众号。

## 3. 一键安装

**双击运行 `install.bat`**，脚本会自动完成：

1. 检查 Node.js 和 Python 是否已安装
2. 安装 Node.js 依赖（`npm install`）
3. 初始化本地 SQLite 数据库
4. 创建 Python 虚拟环境（`venv/`）
5. 安装 Python 依赖（FastAPI + mitmproxy，约需 3-5 分钟）
6. 生成 `.env` 配置文件

安装过程中会弹出命令行窗口，显示进度。**全部显示 [OK] 后按任意键关闭。**

> 如果安装失败，请查看报错信息，最常见原因是 Python 未加入 PATH，
> 重新安装 Python 并勾选 "Add Python to PATH" 即可。

---

## 4. 配置 API Key

安装完成后，用**记事本**打开项目根目录下的 `.env` 文件，
将 `SILICONFLOW_API_KEY` 替换为你的真实密钥：

```env
DATABASE_URL="file:./prisma/dev.db"

SILICONFLOW_API_KEY="sk-你的真实密钥"
SILICONFLOW_LIGHT_MODEL="Qwen/Qwen2.5-7B-Instruct"
SILICONFLOW_HEAVY_MODEL="deepseek-ai/DeepSeek-V3"

INGEST_SECRET="yanzhi-local-secret"

ADMIN_PASSWORD="admin"

NODE_ENV="development"
PORT=3000
```

SiliconFlow API Key 申请地址：[https://cloud.siliconflow.cn](https://cloud.siliconflow.cn)（注册后有免费额度）

> `INGEST_SECRET` 是本机内部通信的验证密钥，可保持默认值不变。

## 5. 启动服务

**双击运行 `start.bat`**，会自动打开三个窗口：

| 窗口名 | 服务 | 端口 |
|--------|------|------|
| 燕知 AI服务 | FastAPI AI 分类 | 8001（内部） |
| 燕知 拦截器 | mitmproxy 代理 | 8080 |
| 当前窗口 | Next.js 网页 | 3000 |

等待当前窗口出现以下字样后，服务启动完成：

```
✓ Ready in xxxms
○ Local:  http://localhost:3000
```

此时打开浏览器访问 [http://localhost:3000](http://localhost:3000) 即可看到网页。

## 6. 配置微信代理

**每次启动服务后**，需要让微信的流量经过 mitmproxy，才能被拦截。

推荐在**微信内部**单独设置代理，而不是开启 Windows 系统代理——这样 VPN 和其他软件完全不受影响。

**在微信 PC 版内设置：**

1. 微信左下角 → **更多** → **设置**
2. 点击 **通用** → **代理**
3. 选择 **手动设置代理**
4. 类型：`HTTP`，地址：`127.0.0.1`，端口：`8080`
5. 点击 **确定**

> 这样只有微信走 mitmproxy，浏览器、VPN、其他软件均不受影响。
> 不采集时可将微信代理改回"系统代理"或"不使用代理"。

## 7. 安装 mitmproxy CA 证书（首次必做）

**仅需操作一次。**

mitmproxy 需要一个受信任的根证书才能解密 HTTPS 流量。

1. 先临时开启 Windows 系统代理（设置 → 网络和 Internet → 代理 → 手动代理 → `127.0.0.1:8080`）
2. 用浏览器（Edge/Chrome）访问：[http://mitm.it](http://mitm.it)
3. 点击 **Windows** 下的证书下载
4. 双击下载的 `.p12` 文件
5. 按向导安装，存储位置选择 **受信任的根证书颁发机构**

安装完成后，mitmproxy 即可解密微信 HTTPS 流量。

## 8. 使用方法

完成以上所有配置后，日常使用非常简单：

1. **双击 `start.bat`** — 启动所有服务
2. **确认微信 PC 版代理已设置**（通用 → 代理 → 手动 → `127.0.0.1:8080`）
3. **打开微信 PC 版**，点开订阅号消息列表，滚动浏览

"燕知 拦截器"窗口会实时显示捕获到的文章：

```
2026-03-07 12:00:00 [interceptor] 从 mp.weixin.qq.com 捕获到 5 篇文章
2026-03-07 12:00:00 [interceptor]   [北京大学] 北大讲座通知 | ...
2026-03-07 12:00:00 [interceptor]   [PKU就业] 2026届春季招聘会...
2026-03-07 12:00:00 [interceptor] 推送 5 篇文章 → 入库 4, 跳过 1（已存在）
```

4. **打开浏览器访问 [http://localhost:3000](http://localhost:3000)** 查看已分类整理的文章

### 不采集时

关闭三个窗口（或直接关闭终端），并将微信代理改回**不使用代理**（避免微信无法联网）。

## 9. 故障排查

### install.bat 报错：未找到 Python

原因：Python 安装时没有勾选 "Add Python to PATH"。

解决：
1. 在开始菜单搜索 "Python"，右键 → 修改/修复
2. 找到 "Add Python to PATH" 选项并启用
3. 重新运行 install.bat

### start.bat 启动后访问 localhost:3000 无法打开

等待约 10-20 秒，Next.js 需要时间编译，出现 `✓ Ready` 字样后再访问。

### 拦截器窗口没有捕获到文章

**情形 1：证书未安装**
确认已按第 7 步安装 mitmproxy CA 证书。可以用浏览器测试：
- 系统代理开启状态下，浏览器访问 `https://www.baidu.com`
- 如果能正常打开，说明证书安装成功
- 如果提示证书错误，说明证书未安装或需要重装

**情形 2：微信 PC 版未走系统代理**
部分版本的微信 PC 会绕过系统代理。解决方法：在微信 PC 版内手动设置代理：
- 微信左下角 **更多** → **设置** → **通用** → **代理** → **手动设置代理**
- 类型选 HTTP，地址 `127.0.0.1`，端口 `8080`

**情形 3：需要刷新微信内容**
mitmproxy 只捕获经过代理的新请求。打开微信 PC 版后，需要**手动点进订阅号消息列表**，让微信发起网络请求，才能被拦截。

### AI 分类不生效（文章分类全是"其他"）

检查 AI 服务窗口是否有报错。常见原因：

- `SILICONFLOW_API_KEY` 未填写或填写错误 → 打开 `.env` 检查
- SiliconFlow 账户余额不足 → 登录 [cloud.siliconflow.cn](https://cloud.siliconflow.cn) 查看

手动测试 AI 服务：
```powershell
curl -X POST http://localhost:8001/classify `
  -H "Content-Type: application/json" `
  -d '{"title":"北大讲座通知","summary":"","content":""}'
```
如果返回 JSON 分类结果则 AI 服务正常。

### 端口被占用

如果提示端口已被占用，关闭所有窗口，在命令行中运行：

```powershell
# 查找并关闭占用 3000 端口的进程
netstat -ano | findstr :3000
taskkill /PID <上面显示的PID> /F

# 同理处理 8001 和 8080
```

## 10. 日常维护

### 更新代码

```powershell
git pull
npm install            # 如果有新的 Node 依赖
venv\Scripts\pip.exe install -r requirements.txt   # 如果有新的 Python 依赖
npx prisma migrate dev # 如果有数据库变更
```

### 查看数据库

```powershell
npx prisma studio
```

浏览器打开 `http://localhost:5555`，可以可视化查看和编辑数据库内容。

### 清空数据重新开始

```powershell
del prisma\dev.db
npx prisma migrate dev --name init
```

### 修改收录的公众号

无需任何配置。只要你的微信账号订阅了某个公众号，刷新订阅号消息时该公众号的推文就会自动被捕获。取消订阅则不再收录，但已入库的文章不会删除。
