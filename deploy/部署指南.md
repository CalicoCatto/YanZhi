## 1. 服务器初始化

SSH 登录服务器后，依次执行以下命令。

### 1.1 更新系统

```bash
apt-get update && apt-get upgrade -y
```

### 1.2 安装 Node.js 20

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt-get install -y nodejs
node -v  # 确认版本 >= 20
```

### 1.3 安装 Python 3、Nginx、pm2

```bash
apt-get install -y python3 python3-pip python3-venv nginx
npm install -g pm2
```

### 1.4 配置 Swap（防内存 OOM）

```bash
fallocate -l 1G /swapfile
chmod 600 /swapfile
mkswap /swapfile && swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab
```

验证：

```bash
free -h  # 应看到 Swap: 1.0G
```

## 2. 上传代码

### 方法 A：从 GitHub 拉取（推荐）

```bash
# 在服务器上执行
git clone git@github.com:CalicoCatto/YanZhi.git /var/www/yanzhi
```

> 如果服务器没有 GitHub SSH 密钥，先运行 `ssh-keygen` 生成，再将公钥添加到 GitHub → Settings → SSH keys。

### 方法 B：本地 rsync 上传

```bash
# 在本地执行
rsync -avz \
  --exclude node_modules \
  --exclude .next \
  --exclude 'prisma/*.db' \
  /home/calico_catto/projects/YanZhi/ \
  root@你的服务器IP:/var/www/yanzhi/
```

## 3. 配置环境变量

```bash
cd /var/www/yanzhi
cp .env.example .env
nano .env
```

填写以下内容（将占位符替换为真实值）：

```env
DATABASE_URL="file:./prisma/prod.db"

SILICONFLOW_API_KEY="sk-你的SiliconFlow密钥"
SILICONFLOW_LIGHT_MODEL="Qwen/Qwen2.5-7B-Instruct"
SILICONFLOW_HEAVY_MODEL="deepseek-ai/DeepSeek-V3"

CRAWL_INTERVAL_HOURS=1
CRAWL_SLEEP_START=1
CRAWL_SLEEP_END=7

ADMIN_PASSWORD="设置一个强密码"

NODE_ENV="production"
PORT=3000
```

## 4. 构建前端

```bash
cd /var/www/yanzhi

# 安装 Node 依赖
npm install

# 初始化生产数据库
npx prisma migrate deploy

# 构建 Next.js（约1-2分钟）
npm run build
```

构建成功后会显示各路由的编译结果，无报错即为正常。

## 5. 安装 Python 爬虫依赖

```bash
cd /var/www/yanzhi/crawler

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

deactivate
```

### 修改 pm2 配置使用虚拟环境

编辑 `deploy/ecosystem.config.js`，将爬虫的 Python 解释器指向虚拟环境：

```js
{
  name: 'yanzhi-crawler',
  script: 'crawler/main.py',
  // 将 'python3' 改为虚拟环境路径
  interpreter: '/var/www/yanzhi/crawler/venv/bin/python3',
  ...
}
```

## 6. 配置 Nginx

### 6.1 在主配置添加限流区域

```bash
nano /etc/nginx/nginx.conf
```

在 `http {` 块内（`include` 行前面）添加：

```nginx
limit_req_zone $binary_remote_addr zone=api_limit:10m rate=30r/m;
```

### 6.2 复制站点配置

```bash
cp /var/www/yanzhi/deploy/nginx.conf /etc/nginx/sites-available/yanzhi

# 如果有域名，编辑 server_name
nano /etc/nginx/sites-available/yanzhi
# 将 server_name _; 改为 server_name 你的域名.com;

# 启用站点，禁用默认站点
ln -s /etc/nginx/sites-available/yanzhi /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
```

### 6.3 验证并重载

```bash
nginx -t          # 检查配置语法
systemctl reload nginx
```

## 7. 启动服务

```bash
cd /var/www/yanzhi

# 启动所有服务（Next.js + 爬虫）
pm2 start deploy/ecosystem.config.js

# 查看运行状态
pm2 status
```

正常输出示例：

```
┌────┬──────────────────┬─────────┬───────┬──────────┐
│ id │ name             │ status  │ cpu   │ memory   │
├────┼──────────────────┼─────────┼───────┼──────────┤
│ 0  │ yanzhi-web       │ online  │ 0.1%  │ 120 MB   │
│ 1  │ yanzhi-crawler   │ online  │ 0.0%  │ 80 MB    │
└────┴──────────────────┴─────────┴───────┴──────────┘
```

### 设置开机自启

```bash
pm2 save
pm2 startup
# 按照命令输出的提示执行对应命令（通常是 systemctl enable pm2-root）
```

## 9. 绑定域名与 SSL（可选）

### 9.1 在阿里云控制台

将域名的 A 记录解析到服务器 IP。

### 9.2 申请免费 SSL 证书

```bash
apt-get install -y certbot python3-certbot-nginx
certbot --nginx -d 你的域名.com -d www.你的域名.com
```

Certbot 会自动修改 Nginx 配置并添加 HTTPS 跳转，证书 90 天自动续期。

## 11. 故障排查

### 网站打不开

```bash
pm2 status              # 确认进程是否 online
systemctl status nginx  # 确认 Nginx 是否运行
curl localhost:3000     # 绕过 Nginx 直接测试
```

### 爬虫不抓取数据

```bash
pm2 logs yanzhi-crawler  # 查看错误信息

# 手动测试
cd /var/www/yanzhi/crawler
source venv/bin/activate
python3 main.py --test
```

常见原因：
- 搜狗反爬限制 → 等待一段时间后重试，或检查 User-Agent 配置
- SiliconFlow API Key 无效 → 检查 `.env` 中的 `SILICONFLOW_API_KEY`
- 数据库权限问题 → `ls -la prisma/prod.db`，确认文件可读写

### 内存不足（OOM）

```bash
free -h        # 查看内存和 Swap 使用情况
pm2 monit      # 查看各进程内存占用

# 如 Swap 不够，增加到 2GB
swapoff /swapfile
fallocate -l 2G /swapfile
mkswap /swapfile && swapon /swapfile
```

### 数据库损坏

```bash
# 备份当前数据库
cp prisma/prod.db prisma/prod.db.bak

# 检查数据库完整性
sqlite3 prisma/prod.db "PRAGMA integrity_check;"

# 重新初始化（会丢失数据）
rm prisma/prod.db
npx prisma migrate deploy
pm2 restart yanzhi-web
```

## 架构总览

```
用户浏览器
    ↓ HTTP/HTTPS
Nginx（80/443）
    ↓ 反向代理
Next.js（3000端口）
    ↓ Prisma ORM
SQLite 数据库（prisma/prod.db）
    ↑ 写入
Python 爬虫（APScheduler，每小时整点）
    ↑ AI 分析
SiliconFlow API（Qwen2.5-7B / DeepSeek-V3）
    ↑ 抓取
搜狗微信搜索（60+ 北大公众号）
```
