## 1. 架构总览

```
Windows PC（用户电脑，登录微信 PC 版）
  WeChat PC — 正常使用，订阅号消息正常接收
       ↓ HTTPS 流量（系统代理 → mitmproxy:8080）
  windows/interceptor.py（mitmproxy addon）
       ↓ 解析到文章数据后
  POST https://你的域名/api/ingest（带 INGEST_SECRET）
       ↓
阿里云服务器
  Nginx（80/443）→ Next.js（3000端口）
  /api/ingest 路由：验证 secret → 去重 → 调 AI 服务
       ↓ http://127.0.0.1:8001/classify（内部）
  server/api_server.py（FastAPI，仅内网，端口 8001）
  AI 两级分类（SiliconFlow：Qwen2.5-7B + DeepSeek-V3）
       ↓
  SQLite 数据库（prisma/prod.db）
       ↓
  Next.js 前端展示
```

**内存分配参考（2GB 服务器）**

| 组件 | 预计占用 |
|------|---------|
| 系统 + sshd | ~300 MB |
| Nginx | ~10 MB |
| Next.js (production) | ~350 MB |
| Python AI 服务 | ~150 MB |
| SQLite | ~10 MB |
| 合计 | ~820 MB |
| Swap 缓冲 | 1024 MB |

## 2. 服务器初始化

SSH 登录服务器后，依次执行以下命令。

### 2.1 更新系统

```bash
apt-get update && apt-get upgrade -y
```

### 2.2 安装 Node.js 20

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt-get install -y nodejs
node -v  # 确认版本 >= 20
```

### 2.3 安装 Python 3、Nginx、pm2

```bash
apt-get install -y python3 python3-pip python3-venv nginx
npm install -g pm2
```

### 2.4 配置 Swap（防内存 OOM）

```bash
fallocate -l 1G /swapfile
chmod 600 /swapfile
mkswap /swapfile && swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab
```

验证：

```bash
free -h  # 应看到 Swap: 1.0G
```

## 3. 上传代码

### 方法 A：从 GitHub 拉取（推荐）

```bash
git clone git@github.com:CalicoCatto/YanZhi.git /var/www/yanzhi
```

> 如果服务器没有 GitHub SSH 密钥，先运行 `ssh-keygen` 生成，再将公钥添加到 GitHub → Settings → SSH keys。

### 方法 B：本地 rsync 上传

```bash
# 在本地执行
rsync -avz \
  --exclude node_modules \
  --exclude .next \
  --exclude 'prisma/*.db' \
  /home/calico_catto/projects/YanZhi/ \
  root@你的服务器IP:/var/www/yanzhi/
```

## 4. 配置环境变量

```bash
cd /var/www/yanzhi
cp .env.example .env
nano .env
```

填写以下内容（将占位符替换为真实值）：

```env
DATABASE_URL="file:./prisma/prod.db"

SILICONFLOW_API_KEY="sk-你的SiliconFlow密钥"
SILICONFLOW_LIGHT_MODEL="Qwen/Qwen2.5-7B-Instruct"
SILICONFLOW_HEAVY_MODEL="deepseek-ai/DeepSeek-V3"

# Windows 端推送文章时使用的共享密钥（自行设置随机字符串）
INGEST_SECRET="设置一个随机字符串，例如：$(openssl rand -hex 24)"

ADMIN_PASSWORD="设置一个强密码"

NODE_ENV="production"
PORT=3000
```

> `INGEST_SECRET` 需要与 Windows 端 `interceptor.py` 中的配置保持一致。

## 6. 安装 Python AI 服务依赖

```bash
cd /var/www/yanzhi

# 在 server/ 目录下创建虚拟环境
python3 -m venv server/venv
source server/venv/bin/activate

# 安装依赖（fastapi, uvicorn, httpx, python-dotenv）
```

验证 AI 服务能否启动：

```bash
source server/venv/bin/activate
cd /var/www/yanzhi
PYTHONPATH=/var/www/yanzhi/server uvicorn server.api_server:app --host 127.0.0.1 --port 8001 &
sleep 2
curl http://127.0.0.1:8001/health  # 应返回 {"ok":true}
kill %1
deactivate
```

## 7. 配置 Nginx

### 7.1 在主配置添加限流区域

```bash
nano /etc/nginx/nginx.conf
```

在 `http {` 块内（`include` 行前面）添加：

```nginx
limit_req_zone $binary_remote_addr zone=api_limit:10m rate=30r/m;
```

### 7.2 复制站点配置

```bash
cp /var/www/yanzhi/deploy/nginx.conf /etc/nginx/sites-available/yanzhi

# 如果有域名，编辑 server_name
nano /etc/nginx/sites-available/yanzhi
# 将 server_name _; 改为 server_name 你的域名.com;

# 启用站点，禁用默认站点
ln -s /etc/nginx/sites-available/yanzhi /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
```

### 7.3 验证并重载

```bash
nginx -t          # 检查配置语法
systemctl reload nginx
```

## 10. 绑定域名与 SSL（可选）

### 10.1 在阿里云控制台

将域名的 A 记录解析到服务器 IP。

### 10.2 申请免费 SSL 证书

```bash
apt-get install -y certbot python3-certbot-nginx
certbot --nginx -d 你的域名.com -d www.你的域名.com
```

Certbot 会自动修改 Nginx 配置并添加 HTTPS 跳转，证书 90 天自动续期。

## 11. Windows 端配置

文章数据由 Windows 电脑上的微信 PC 版产生，通过本地 mitmproxy 代理拦截后推送到服务器。

### 11.1 安装依赖

```powershell
# 在 Windows 命令行或 PowerShell 中执行
pip install mitmproxy requests
```

### 11.2 配置 interceptor.py

编辑项目中的 `windows/interceptor.py`，修改顶部的两个配置项：

```python
# 改为你的实际服务器地址
SERVER_URL = "https://你的域名/api/ingest"

# 与服务器 .env 中 INGEST_SECRET 完全一致
INGEST_SECRET = "你设置的随机字符串"
```

也可以通过环境变量传入，避免硬编码：

```powershell
set YANZHI_SERVER_URL=https://你的域名/api/ingest
set INGEST_SECRET=你设置的随机字符串
```

### 11.3 启动拦截器

```powershell
cd windows
mitmdump -s interceptor.py --listen-port 8080
```

### 11.4 设置系统代理

在 Windows 中：**设置 → 网络和 Internet → 代理 → 手动设置代理**

- 开关：**开启**
- 地址：`127.0.0.1`
- 端口：`8080`

### 11.5 安装 mitmproxy CA 证书（首次必做）

系统代理设置完成后，用浏览器访问：

```
http://mitm.it
```

点击对应系统的证书下载并安装，安装时选择"受信任的根证书颁发机构"。

> 证书只需安装一次，之后每次启动 mitmdump 即可直接使用。

### 11.6 验证拦截效果

启动后打开微信 PC 版，点开任意一条订阅号消息列表，终端中应出现类似输出：

```
2026-03-07 12:00:00 [interceptor] 从 mp.weixin.qq.com 捕获到 3 篇文章
2026-03-07 12:00:00 [interceptor]   [北京大学] 北大讲座 | XXX主题报告
2026-03-07 12:00:00 [interceptor]   [PKU学生] 关于五四奖学金评选的通知
2026-03-07 12:00:00 [interceptor] 推送 3 篇文章 → 服务器接收 3, 入库 3, 跳过 0
```

### 11.7 日常使用

每次需要采集文章时：

1. 启动 `mitmdump -s interceptor.py --listen-port 8080`
2. 确认系统代理已开启
3. 打开微信，正常浏览订阅号消息即可，文章自动入库
4. 不需要采集时，关闭终端并关闭系统代理

## 13. 故障排查

### 网站打不开

```bash
pm2 status              # 确认进程是否 online
systemctl status nginx  # 确认 Nginx 是否运行
curl localhost:3000     # 绕过 Nginx 直接测试
```

### ingest 接口返回 401

原因：`Authorization` header 与服务器 `.env` 中的 `INGEST_SECRET` 不一致。

```bash
grep INGEST_SECRET /var/www/yanzhi/.env  # 查看服务器端配置
```

确认 Windows 端 `interceptor.py` 中的 `INGEST_SECRET` 与之完全相同。

### Windows 端没有捕获到文章

常见原因及排查步骤：

1. **CA 证书未安装** — 重新访问 `http://mitm.it` 安装证书
2. **系统代理未开启** — 检查 Windows 代理设置
3. **微信 PC 版使用了独立代理** — 部分版本微信会绕过系统代理，
   尝试在微信设置中手动设置代理为 `127.0.0.1:8080`
4. **微信 API 响应结构变化** — 查看 mitmdump 输出，确认是否有
   `weixin.qq.com` 的请求被捕获，如有捕获但未提取到文章，
   说明微信 JSON 结构有变化，需要更新 `interceptor.py` 中的字段名

### AI 服务无响应（ingest 入库但分类为"其他"）

```bash
pm2 logs yanzhi-ai    # 查看 AI 服务错误
curl http://127.0.0.1:8001/health  # 确认服务在线

# 手动测试分类
curl -X POST http://127.0.0.1:8001/classify \
  -H "Content-Type: application/json" \
  -d '{"title":"北大讲座通知","summary":"","content":""}'
```

常见原因：
- SiliconFlow API Key 无效 → 检查 `.env` 中的 `SILICONFLOW_API_KEY`
- Python 虚拟环境路径错误 → 检查 `deploy/ecosystem.config.js` 中的 `interpreter`

### 内存不足（OOM）

```bash
free -h     # 查看内存和 Swap 使用情况
pm2 monit   # 查看各进程内存占用

# 如 Swap 不够，增加到 2GB
swapoff /swapfile
fallocate -l 2G /swapfile
mkswap /swapfile && swapon /swapfile
```

### 数据库损坏

```bash
# 备份当前数据库
cp prisma/prod.db prisma/prod.db.bak

# 检查数据库完整性
sqlite3 prisma/prod.db "PRAGMA integrity_check;"

# 重新初始化（会丢失数据）
rm prisma/prod.db
npx prisma migrate deploy
pm2 restart yanzhi-web
```
